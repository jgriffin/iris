DetectionLayer · quad-rendering reference

The same tilted Vision-rectangle fixture rendered two ways. A rectangle detection comes back as four oriented corner keypoints (topLeft → topRight → bottomRight → bottomLeft), but the old overlay drew only the axis-aligned boundingBox envelope — an upright box that misrepresents a rotated quad. The new DetectionLayer.quadCorners(of:) path connects the actual corners. Coordinates apply the same Y-flip the converter does (y_screen = (1 − y_norm) · height, Vision bottom-left origin → screen top-left). Open in a browser; no build step.

Why this exists: per CLAUDE.md's "favorite pattern", visual artifacts get caught at the artifact level. The point of the side-by-side is that the spatial-honesty win is obvious at a glance — the dashed upright box clearly does not describe a rotated rectangle.
