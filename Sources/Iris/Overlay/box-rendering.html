DetectionLayer · box-rendering reference

Static visual reference for the boxes DetectionLayer draws, rendered with OverlayStyle.default. The numbers in OverlayStyle.swift (stroke color, stroke width, label font, backplate opacity) are mirrored as CSS variables at the top of this file — keep them in lockstep with any change to the defaults. Each section shows the container, the post-aspect-fit videoRect, and a few synthetic boxes drawn against it. Open in a browser; no build step.

Why this exists: per CLAUDE.md's "favorite pattern", visual artifacts get caught at the artifact level — same way unit tests catch logic bugs without running the full app. If the boxes here look wrong, the boxes in the SwiftUI Canvas will look the same wrong. Fix the style here, run the test suite, then run the demo.
