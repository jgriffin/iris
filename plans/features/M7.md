## Scope / intent

Turn playback into a **dataset-curation loop**: scrub a video (or a recording),
spot a frame the detector got wrong, **flag it** for the training set, and later
**extract** the flagged frames тАФ provenance-bearing PNGs, **no sidecar** тАФ into a
dataset library on disk. Annotation proper (and any training-format export)
happens *separately* in an external tool; M7's job is to **find, mark, persist,
and extract** the right frames without duplicating work or committing to a
training format prematurely.

This delivers BRIEF [┬з6 `IrisDataset`](../BRIEF.md) and the
**M7 тАФ Dataset** milestone (one-tap frame extraction from playback; training-format
export deferred until a pipeline names it).

**The design turns on one observation:** a playback frame is *already* uniquely
addressable as `(asset, presentation-timestamp)`, and `seek(to:)` with `.zero`
tolerance round-trips a stored PTS back to the **exact same frame**
([`PlaybackSource.seek`](../../Sources/Iris/Playback/PlaybackSource.swift)). So
flagging never needs to touch pixels тАФ it records a tiny address + metadata тАФ
and extraction is a deferred, headless batch that re-seeks each address and
decodes on demand. `PlaybackSource`'s default `TaskTickDriver` already exists
"for headless use (no view, e.g. dataset processing)"
([class doc](../../Sources/Iris/Playback/PlaybackSource.swift)) тАФ M7 is the
consumer it was built for.

**Two clocks, deliberately separated:**

| Phase      | Cost          | When        | Writes        |
| ---------- | ------------- | ----------- | ------------- |
| Flagging   | cheap, instant | while scrubbing | metadata only |
| Extraction | expensive      | deferred, batch | image (PNG) only |

This is why we *don't* write frame pixels in the hot loop тАФ exactly the
constraint that motivated the milestone.

## Decisions locked (2026-05-28)

- **Canonical frame address тЖТ `FrameRef = (AssetFingerprint, pts)`.** The
  presentation timestamp (PTS) is the address; stored as an exact `CMTime`
  (`{value: Int64, timescale: Int32}`, rational тАФ **no float drift**). Frame
  *index* is rejected тАФ variable frame rate breaks it; PTS + `.zero`-tolerance
  seek is reproducible across reloads.
- **Asset identity тЖТ name-INDEPENDENT content fingerprint, not URL.**
  `AssetFingerprint.id` = hash of `byteSize` + `durationSeconds` + `headHash`
  (head-hash of the first ~1 MB is now **mandatory**, not optional). Filename is
  **dropped from identity** тАФ kept only as a display field. This makes the
  fingerprint **rename-stable** (reload a renamed video тЖТ same flags) *and*
  **edit-sensitive** (clipping from the middle shifts duration or head-hash тЖТ a
  genuinely different asset). Rejected: bare `url.absoluteString` (breaks on
  move); filename-inclusive fingerprint (breaks on rename тАФ *this corrects an
  earlier draft of this bullet that claimed the filename-inclusive form "survived
  renames"; it didn't*); full-file SHA-256 (too slow on large video тАФ fights
  "robust against loads of video").
- **Flag store тЖТ per-asset JSON, library-side, base dir injected.** `FlagStore`
  is `@MainActor @Observable` (mirrors
  [`RecentVideos`](../../Apps/Shared/RecentVideos.swift)) but persists to
  `<baseDir>/iris-dataset/flags/<fingerprint.id>.json` (one file per asset,
  `JSONEncoder`, atomic, `.v1`-versioned). The library does **not** hardcode the
  app's Documents dir тАФ the app injects `baseDir` (decouples library from app
  sandbox; keeps Iris's "no hardcoded paths" discipline). This is Iris's **first
  library-side on-disk persistence**.
- **No sidecar, no COCO in M7 тЖТ the dataset's filenames are the ledger.**
  A flag means "this frame deserves a second look," NOT a ground-truth label тАФ
  and since multiple models get tried, baking one model's verdict
  (`modelID`/score/"was wrong") into a per-image annotation is false precision.
  So extraction writes **provenance-bearing PNGs only, no sidecar.** Export
  filename = `<sourceNameHash>_<fingerprintID>_<ptsMillis>.png`, where
  `sourceNameHash` is a short hash of the source filename тАФ a cosmetic,
  collision-tolerant grouping prefix (human-visible if not readable,
  recomputable from a directory listing). **Dedup keys on the
  `_<fingerprintID>_<ptsMillis>` suffix, not the full name**, so a renamed
  source still counts as already-exported. The principle: *the "already
  exported?" answer must live WITH the dataset (its filenames), because the
  dataset can be lost or moved independently of the app's `FlagStore`* тАФ a
  sidecar/manifest can go stale relative to the data; filenames cannot. Two
  questions, two homes тАФ "what did I flag?" тЖТ `FlagStore`; "what's in the
  dataset?" тЖТ the dataset's filenames. Real annotation (and any training-format
  export) happens later in an external tool; **M7 commits to no training format.**
  тЖТ [`DECISIONS.md`](../DECISIONS.md) (2026-05-28 sidecar reframe).
- **Output тЖТ app-managed Documents folder.** Dataset lands under
  `<Documents>/iris-dataset/` тАФ already browsable in Files.app via the
  file-sharing work merged on `demo-sim-runnable`
  (`8a9e9c1`). `DatasetSink` is a protocol with `FolderDatasetSink` as the v1
  impl; iCloud/S3 sinks (BRIEF ┬з6) are left as protocol room, not built.
- **Dedup тЖТ deterministic naming + sink existence check.** Extracted frame тЖТ
  `<sourceNameHash>_<fingerprintID>_<ptsMillis>.png` (no sidecar). Same frame тЗТ
  same `_<fingerprintID>_<ptsMillis>` suffix, so "already captured?" is a
  `sink.contains(ref)` **suffix** match against the dataset's filenames тАФ no
  fuzzy matching, rename-stable. Extraction is therefore **resumable** and
  re-runs are cheap.
- **Scope = playback context.** Deferred extraction relies on re-seeking the
  PTS, which only playback affords. Live-capture one-tap (immediate
  pixel-buffer snapshot, a different path) is a **follow-on**, not M7 v1 тАФ see
  Opens.

## Phases

### P1 тАФ `FrameRef` + `AssetFingerprint` + `FlagStore` (library core, no UI)
New folder [`Sources/Iris/Dataset/`](../../Sources/Iris/Dataset/) (doesn't exist yet):
- `FrameRef.swift` тАФ `public struct FrameRef: Sendable, Hashable, Codable`
  wrapping `AssetFingerprint` + a `Codable` `CMTime` (`{value, timescale}`; AVF's
  `CMTime` isn't `Codable`, so a thin wrapper).
- `AssetFingerprint.swift` тАФ the content fingerprint + `static func
  compute(url:) async throws` (file size via `FileManager` attrs, duration via
  `AVAsset.load(.duration)`, optional head-hash). Exposes a stable `id: String`
  used as the filesystem-safe naming key.
- `FrameFlag.swift` тАФ the record: `FrameRef` + `detections: [Detection]`,
  `modelID: String?`, `confidenceThreshold: Double?`, `reason: FlagReason`
  (`.wrong` / `.nearMiss` / `.other` тАФ BRIEF's "this was wrong"/"near-miss"
  affordances), `note: String?`, `flaggedAt: Date`.
- `FlagStore.swift` тАФ `@MainActor @Observable`; `flags(for:)`, `isFlagged(_:)`,
  `toggle(_:)`, `add/remove`; per-asset JSON load/save under injected `baseDir`.
- **`Detection` (+ nested skeleton/readout) must gain `Codable`** тАФ no `Codable`
  exists in `Sources/Iris/` today. Either conform `Detection` directly or
  introduce a `DetectionRecord` DTO; pick whichever keeps the public type clean.
- **Tests** ([`Tests/IrisTests/Dataset/`](../../Tests/IrisTests/)): fingerprint
  stability across a path move (copy a fixture clip to a new name, assert same
  `id`); PTS round-trips a `FrameRef` exactly through encode/decode; flag
  survives store reload; `Detection` Codable round-trip.

### P2 тАФ Flagging UI: bookmark toggle, timeline markers, flagged-frames panel
- **Bookmark toggle** on the scrubber тАФ flags/unflags the current frame.
  Built-in thin control (per the M4 doctrine: *UI lives outside Iris, ship a thin
  built-in for the simple case* тАФ [`m4_tuning_design_shape`]). Lives next to
  [`Scrubber.swift`](../../Sources/Iris/Playback/Scrubber.swift) /
  [`ScrubberModel.swift`](../../Sources/Iris/Playback/ScrubberModel.swift).
- **Timeline markers** тАФ render flagged PTSs as ticks/bookmarks along the
  scrubber track so flagged positions are visible while scrubbing.
- **Flagged-frames side panel** тАФ list of the current asset's flags; tap тЖТ
  `seek(to: pts)` (**jump-to-flag**). Mirror the macOS demo's
  `NavigationSplitView` + `List(selection:)` sidebar
  ([`Apps/IrisDemo-macOS/ContentView.swift`](../../Apps/IrisDemo-macOS/ContentView.swift));
  iOS adapts (sheet/popover on iPhone, sidebar on iPad).
- Wire the current `FrameRef` from the coordinator's current asset fingerprint +
  the live `Frame.timestamp` / scrubber position.

### P3 тАФ `DatasetSink` + headless extractor + dedup (no sidecar)
- `DatasetSink.swift` тАФ protocol: `func contains(_ ref: FrameRef) -> Bool` +
  `func write(image:for:) async throws` (image only тАФ no sidecar param).
- `FolderDatasetSink.swift` тАФ writes
  `<Documents>/iris-dataset/frames/<sourceNameHash>_<fingerprintID>_<ptsMillis>.png`;
  `contains` matches on the `_<fingerprintID>_<ptsMillis>` **suffix** of the
  existing filenames (rename-stable dedup ledger). v1 impl.
- `DatasetBuilder.swift` тАФ headless batch: for an asset's flags, build
  `PlaybackSource(url:)` (default `TaskTickDriver`), `seek(to: pts)` тЖТ
  `emitOneShotFrame` тЖТ grab the `Frame`, **skip if `sink.contains(ref)`**,
  else encode `pixelBuffer` тЖТ PNG and write the image. Resumable.
- **Image encoding** тАФ no reusable Overlay CIImage path existed in
  `Sources/Iris/`, so P3 adds a small CoreImage-only `PixelBufferPNGEncoder`
  (`CVPixelBuffer` тЖТ PNG). (Resolves the prior "reuse an Overlay CIImage path"
  open тАФ there was nothing to reuse.)
- Tests: extract a small flag set from a fixture clip; assert deterministic
  filenames, suffix-dedup skip on re-run (incl. a renamed-source case), encoded
  PNG decodes.

### P4 тАФ Frame export sweep (`FrameExporter`): keep `frames/` filling up automatically as you flag
A concrete, nearer-term thing тАФ the **auto frame-writer** that drives the P3
extraction loop on a schedule. Distinct from the still-deferred
`DatasetExporter` (training-FORMAT conversion тАФ see the follow-on note below).

- **Library `FrameExporter`** тАФ given a set of candidate video URLs, for each:
  compute the [`AssetFingerprint`](../../Sources/Iris/Dataset/AssetFingerprint.swift),
  find that asset's flags not yet on disk, and extract them via the existing
  [`DatasetBuilder`](../../Sources/Iris/Dataset/DatasetBuilder.swift).
  - **Resumable** тАФ a re-run skips anything `sink.contains` already has (the same
    suffix-dedup ledger from P3 тАФ the dataset's own filenames).
  - **Interruptible** тАФ cooperative `Task` cancellation checked between assets and
    between frames; cancel is cheap, and the next run picks up where it left off.
- **Source of videos = [`RecentVideos`](../../Apps/Shared/RecentVideos.swift)** тАФ
  the app resolves its security-scoped bookmarks тЖТ URLs and hands them to the
  library; the library stays ignorant of bookmarks/UserDefaults. Chosen approach
  **(a)** over **(b)** a dedicated unbounded "flagged sources" ledger.
  - **MRU caveat:** `RecentVideos` is capped at 10, so a video flagged but not
    recently opened falls off the list and won't be swept until reopened.
    **(b)** is the noted follow-up if that cap bites.
- **Triggers тАФ app-side, in both demos** (triggers + `RecentVideos` resolution
  are app concerns, not the library's, per Iris doctrine):
  - on **`scenePhase` тЖТ background/inactive** (debounced), in a cancellable `Task`
    that is **cancelled on return to foreground** тАФ the "idle / on close" hook;
  - plus a manual **"Export now"** button in the flagged-frames panel for
    forcing/testing.
  - **On-launch trigger was dropped** (user, 2026-05-28): launching a sweep at
    app-launch contends with the user immediately opening/playing a video (the
    sweep opens headless `PlaybackSource`s). A *delayed*-after-launch sweep was
    floated but parked тАФ not wanted now. Background + manual are the triggers.
- **Unreachable-source tracking** тАФ the sweep reports flagged assets it couldn't
  reach (no matching resolvable source) by `(fingerprintID, displayFilename,
  pendingCount)`, and persists a small **operational**
  `<Documents>/iris-dataset/export-status.json` (last-run counts + unreachable
  list). This is operational telemetry тАФ **NOT** a revival of the rejected
  per-frame provenance sidecar.

**Follow-on, further deferred тАФ the real `DatasetExporter` (format conversion).**
Moving or format-exporting the frame files elsewhere (COCO / YOLO / Pascal-VOC)
stays out: a flag is "look again," not an annotation, and multiple models get
tried, so committing to a training format now is false precision. **Build the
first concrete exporter when a training pipeline exists and names its consumer**
тАФ at which point its format is known rather than guessed. (COCO-80 label reuse
via [`COCOLabels.swift`](../../Sources/Iris/Detection/CoreML/COCOLabels.swift)
stays available for whichever exporter eventually lands.) See
[`DECISIONS.md`](../DECISIONS.md) (2026-05-28 P4 redefinition).

## Opens / risks

- **Capture-context flagging is a separate mechanism.** BRIEF ┬з6 says "both live
  and playback"; M7 v1 is **playback only** because deferred extraction needs
  re-seek. Live capture must snapshot the pixel buffer *at flag time* (the
  real-time write we deliberately avoid on playback). Track as a follow-on once
  the playback loop is proven.
- **PTS exactness vs `currentTime()`.** `Frame.timestamp` is
  `playerItem.currentTime()` at tick time, which may sit slightly off the true
  sample PTS. P1 should decide whether to **snap to the nearest sample PTS** on
  flag so the stored address is canonical; otherwise re-seek + `.zero` tolerance
  may land on an adjacent frame. Validate against a fixture in P1.
- **`Detection` Codable churn.** Self-describing detections (skeleton + readout)
  may make a clean `Codable` conformance awkward тАФ a `DetectionRecord` DTO might
  be the cleaner seam. Decide in P1, not by accident.
- **Fingerprint collisions.** size+duration alone is strong but not unique; the
  head-hash (now **mandatory** тАФ see locked decisions) is the safety valve.
  Confirm it stays cheap enough to keep on by default. *(Resolved direction: the
  fingerprint is name-independent тАФ filename is no longer part of identity.)*
- **`AssetID.raw` vs `AssetFingerprint`.** Today `AssetID.raw = url.absoluteString`
  ([`AssetID.swift`](../../Sources/Iris/AssetID.swift)). M7 introduces a richer
  identity *alongside* it тАФ don't conflate; the fingerprint is dataset-scoped,
  `AssetID` stays the playback-session handle.
- тЬЕ **Image-encoding location тАФ resolved (P3).** No reusable `Overlay`-side
  CIImage path existed in `Sources/Iris/`, so P3 added a small CoreImage-only
  `PixelBufferPNGEncoder`. Nothing to share, so no single-source-of-truth concern.
