# M7 — Dataset: flag frames during playback, extract them for training later

<!-- Working plan. Lifetime ~ this milestone; LOG.md keeps the trail. Status vocab per WORKFLOW.md §"Status trees". -->
_Defined · 2026-05-28_ · **✏️ P1–P4 defined, not started · branch TBD**

## Scope / intent

Turn playback into a **dataset-curation loop**: scrub a video (or a recording),
spot a frame the detector got wrong, **flag it** for the training set, and later
**extract** the flagged frames — image + COCO-shaped sidecar — into a dataset
library on disk. Annotation proper happens *separately* in an external tool; M7's
job is to **find, mark, persist, and extract** the right frames without
duplicating work.

This delivers BRIEF [§6 `IrisDataset`](../BRIEF.md) and the
**M7 — Dataset** milestone (one-tap frame + metadata + COCO sidecar, from
playback).

**The design turns on one observation:** a playback frame is *already* uniquely
addressable as `(asset, presentation-timestamp)`, and `seek(to:)` with `.zero`
tolerance round-trips a stored PTS back to the **exact same frame**
([`PlaybackSource.seek`](../../Sources/Iris/Playback/PlaybackSource.swift)). So
flagging never needs to touch pixels — it records a tiny address + metadata —
and extraction is a deferred, headless batch that re-seeks each address and
decodes on demand. `PlaybackSource`'s default `TaskTickDriver` already exists
"for headless use (no view, e.g. dataset processing)"
([class doc](../../Sources/Iris/Playback/PlaybackSource.swift)) — M7 is the
consumer it was built for.

**Two clocks, deliberately separated:**

| Phase      | Cost          | When        | Writes        |
| ---------- | ------------- | ----------- | ------------- |
| Flagging   | cheap, instant | while scrubbing | metadata only |
| Extraction | expensive      | deferred, batch | image + sidecar |

This is why we *don't* write frame pixels in the hot loop — exactly the
constraint that motivated the milestone.

## Decisions locked (2026-05-28)

- **Canonical frame address → `FrameRef = (AssetFingerprint, pts)`.** The
  presentation timestamp (PTS) is the address; stored as an exact `CMTime`
  (`{value: Int64, timescale: Int32}`, rational — **no float drift**). Frame
  *index* is rejected — variable frame rate breaks it; PTS + `.zero`-tolerance
  seek is reproducible across reloads.
- **Asset identity → content fingerprint, not URL.** `AssetFingerprint` =
  `filename` + `byteSize` + `durationSeconds` (+ optional head-hash of the first
  ~1 MB for collision safety). Survives moves/renames so **reload the same video
  → same flags**. Rejected: bare `url.absoluteString` (breaks on move); full-file
  SHA-256 (too slow on large video — fights "robust against loads of video").
- **Flag store → per-asset JSON, library-side, base dir injected.** `FlagStore`
  is `@MainActor @Observable` (mirrors
  [`RecentVideos`](../../Apps/Shared/RecentVideos.swift)) but persists to
  `<baseDir>/iris-dataset/flags/<fingerprint.id>.json` (one file per asset,
  `JSONEncoder`, atomic, `.v1`-versioned). The library does **not** hardcode the
  app's Documents dir — the app injects `baseDir` (decouples library from app
  sandbox; keeps Iris's "no hardcoded paths" discipline). This is Iris's **first
  library-side on-disk persistence**.
- **Sidecar → per-image, COCO-shaped + merge-exporter.** Each extracted frame
  gets its own small COCO-fragment JSON (predicted detections + capture
  metadata). A separate `COCOExporter` merges them into one canonical
  `annotations.json` on demand. Matches BRIEF §6 ("image + JSON sidecar") and
  keeps extraction incremental + dedup-friendly. Rejected: one growing
  dataset-wide COCO file (concurrent/incremental append + dedup are fragile).
- **Output → app-managed Documents folder.** Dataset lands under
  `<Documents>/iris-dataset/` — already browsable in Files.app via the
  file-sharing work merged on `demo-sim-runnable`
  (`8a9e9c1`). `DatasetSink` is a protocol with `FolderDatasetSink` as the v1
  impl; iCloud/S3 sinks (BRIEF §6) are left as protocol room, not built.
- **Dedup → deterministic naming + sink existence check.** Extracted frame →
  `<fingerprint.id>_<ptsMillis>.png` (+ `.json`). Same frame ⇒ same name, so
  "already captured?" is a `sink.contains(ref)` file/manifest check — no fuzzy
  matching. Extraction is therefore **resumable** and re-runs are cheap.
- **Scope = playback context.** Deferred extraction relies on re-seeking the
  PTS, which only playback affords. Live-capture one-tap (immediate
  pixel-buffer snapshot, a different path) is a **follow-on**, not M7 v1 — see
  Opens.

## Phases

### P1 — `FrameRef` + `AssetFingerprint` + `FlagStore` (library core, no UI)
New folder [`Sources/Iris/Dataset/`](../../Sources/Iris/Dataset/) (doesn't exist yet):
- `FrameRef.swift` — `public struct FrameRef: Sendable, Hashable, Codable`
  wrapping `AssetFingerprint` + a `Codable` `CMTime` (`{value, timescale}`; AVF's
  `CMTime` isn't `Codable`, so a thin wrapper).
- `AssetFingerprint.swift` — the content fingerprint + `static func
  compute(url:) async throws` (file size via `FileManager` attrs, duration via
  `AVAsset.load(.duration)`, optional head-hash). Exposes a stable `id: String`
  used as the filesystem-safe naming key.
- `FrameFlag.swift` — the record: `FrameRef` + `detections: [Detection]`,
  `modelID: String?`, `confidenceThreshold: Double?`, `reason: FlagReason`
  (`.wrong` / `.nearMiss` / `.other` — BRIEF's "this was wrong"/"near-miss"
  affordances), `note: String?`, `flaggedAt: Date`.
- `FlagStore.swift` — `@MainActor @Observable`; `flags(for:)`, `isFlagged(_:)`,
  `toggle(_:)`, `add/remove`; per-asset JSON load/save under injected `baseDir`.
- **`Detection` (+ nested skeleton/readout) must gain `Codable`** — no `Codable`
  exists in `Sources/Iris/` today. Either conform `Detection` directly or
  introduce a `DetectionRecord` DTO; pick whichever keeps the public type clean.
- **Tests** ([`Tests/IrisTests/Dataset/`](../../Tests/IrisTests/)): fingerprint
  stability across a path move (copy a fixture clip to a new name, assert same
  `id`); PTS round-trips a `FrameRef` exactly through encode/decode; flag
  survives store reload; `Detection` Codable round-trip.

### P2 — Flagging UI: bookmark toggle, timeline markers, flagged-frames panel
- **Bookmark toggle** on the scrubber — flags/unflags the current frame.
  Built-in thin control (per the M4 doctrine: *UI lives outside Iris, ship a thin
  built-in for the simple case* — [`m4_tuning_design_shape`]). Lives next to
  [`Scrubber.swift`](../../Sources/Iris/Playback/Scrubber.swift) /
  [`ScrubberModel.swift`](../../Sources/Iris/Playback/ScrubberModel.swift).
- **Timeline markers** — render flagged PTSs as ticks/bookmarks along the
  scrubber track so flagged positions are visible while scrubbing.
- **Flagged-frames side panel** — list of the current asset's flags; tap →
  `seek(to: pts)` (**jump-to-flag**). Mirror the macOS demo's
  `NavigationSplitView` + `List(selection:)` sidebar
  ([`Apps/IrisDemo-macOS/ContentView.swift`](../../Apps/IrisDemo-macOS/ContentView.swift));
  iOS adapts (sheet/popover on iPhone, sidebar on iPad).
- Wire the current `FrameRef` from the coordinator's current asset fingerprint +
  the live `Frame.timestamp` / scrubber position.

### P3 — `DatasetSink` + headless extractor + dedup
- `DatasetSink.swift` — protocol: `func contains(_ ref: FrameRef) -> Bool` +
  `func write(image:sidecar:for:) async throws`.
- `FolderDatasetSink.swift` — writes `<Documents>/iris-dataset/frames/<id>_<ptsMillis>.png`
  + sidecar `.json`; `contains` = file-exists. v1 impl.
- `DatasetBuilder.swift` — headless batch: for an asset's flags, build
  `PlaybackSource(url:)` (default `TaskTickDriver`), `seek(to: pts)` →
  `emitOneShotFrame` → grab the `Frame`, **skip if `sink.contains(ref)`**,
  else encode `pixelBuffer` → PNG and write image + sidecar. Resumable.
- **Image encoding** (YUV-420 `CVPixelBuffer` → PNG) — locate/confirm an
  existing converter (Overlay likely has CIImage plumbing) or add one; sub-task
  of P3.
- Tests: extract a small flag set from a fixture clip; assert deterministic
  filenames, dedup skip on re-run, sidecar contents.

### P4 — COCO sidecar schema + `COCOExporter`
- Finalize the per-image sidecar schema (COCO `images`/`annotations`/`categories`
  fragment carrying the **predicted** detections as *provisional* annotations +
  provenance, since human annotation refines later).
- `COCOExporter.swift` — merge all per-image sidecars in a dataset dir into one
  canonical COCO `annotations.json`. Categories from the detector's label set
  (the YOLO COCO-80 case reuses
  [`COCOLabels.swift`](../../Sources/Iris/Detection/CoreML/COCOLabels.swift);
  general detectors supply their own).
- Leaves room for YOLO / Pascal VOC exporters (BRIEF §6) — not built in M7.

## Owns / out of scope

- **Owns:** `Sources/Iris/Dataset/` (FrameRef, AssetFingerprint, FrameFlag,
  FlagStore, DatasetSink + FolderDatasetSink, DatasetBuilder, COCOExporter);
  `Detection` Codable; thin built-in flagging controls + flagged-frames panel;
  demo wiring of both.
- **Out:** the external annotation tool (frames are extracted for it, not
  annotated in-app); iCloud/S3 `DatasetSink` impls (protocol room only);
  YOLO/Pascal-VOC exporters (COCO canonical only); **live-capture flagging**
  (follow-on, different mechanism — see Opens); multi-detector / "what if?"
  dataset interactions (separate QUESTIONS entries).

## Opens / risks

- **Capture-context flagging is a separate mechanism.** BRIEF §6 says "both live
  and playback"; M7 v1 is **playback only** because deferred extraction needs
  re-seek. Live capture must snapshot the pixel buffer *at flag time* (the
  real-time write we deliberately avoid on playback). Track as a follow-on once
  the playback loop is proven.
- **PTS exactness vs `currentTime()`.** `Frame.timestamp` is
  `playerItem.currentTime()` at tick time, which may sit slightly off the true
  sample PTS. P1 should decide whether to **snap to the nearest sample PTS** on
  flag so the stored address is canonical; otherwise re-seek + `.zero` tolerance
  may land on an adjacent frame. Validate against a fixture in P1.
- **`Detection` Codable churn.** Self-describing detections (skeleton + readout)
  may make a clean `Codable` conformance awkward — a `DetectionRecord` DTO might
  be the cleaner seam. Decide in P1, not by accident.
- **Fingerprint collisions.** filename+size+duration is strong but not unique;
  the optional head-hash is the safety valve. Confirm it's cheap enough to keep
  "on" by default.
- **`AssetID.raw` vs `AssetFingerprint`.** Today `AssetID.raw = url.absoluteString`
  ([`AssetID.swift`](../../Sources/Iris/AssetID.swift)). M7 introduces a richer
  identity *alongside* it — don't conflate; the fingerprint is dataset-scoped,
  `AssetID` stays the playback-session handle.
- **Image-encoding location.** Confirm whether an `Overlay`-side CIImage path can
  be reused for PNG export before adding a new converter (avoid duplicate
  coordinate/format logic — single source of truth).
