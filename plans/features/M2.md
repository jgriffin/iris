# M2 — `IrisDetection` + `IrisOverlay`

**Scope.** iOS-only. Builds detection + overlay subsystems on top of M1's
`Source` protocol and `Frame` stream. A Vision-backed `Detector` runs over
`AsyncStream<Frame>` from `CaptureSession`; results land in a `ResultStore`
keyed by `CMTime`; a SwiftUI `DetectionLayer` (`Canvas` + `TimelineView`) reads
the store at `displayTime` and draws boxes. End-to-end live demo extends the
existing `Apps/IrisDemo-iOS` app. macOS playback parity is M3.

**Locked design.** Phases below realize the locked recommendations in:

- [`explorations/runtime-pipeline-architecture/RECOMMENDATIONS.md`](../../explorations/runtime-pipeline-architecture/RECOMMENDATIONS.md)
- [`explorations/display-pipeline-architecture/RECOMMENDATIONS.md`](../../explorations/display-pipeline-architecture/RECOMMENDATIONS.md)

## Public surface

**`Sources/Iris/Detection/`:**

- `Detector: Sendable` — `func detect(_ frame: Frame) async throws -> [Detection]`
- `Detection` — bounding box, label, confidence, optional keypoints/mask, source model ID
- `DetectorPipeline` — chains or parallelizes multiple detectors per frame
- Vision adapter(s) — conform to `Detector` by wrapping Vision's Swift `VNRequest` API

**`Sources/Iris/Overlay/`:**

- `DetectionLayer: View` — SwiftUI `Canvas` overlay with `TimelineView` redraw, `.drawingGroup()`, `.allowsHitTesting(false)`
- `ResultStore` — `@MainActor @Observable`; sorted ring buffer keyed by `CMTime`; binary-search `lookup(at:)` with staleness thresholds
- `TimestampedDetections` — `CMTime` + `[Detection]` pair; `Sendable, Hashable`
- `NormalizedGeometryConverting` — protocol with `viewRect(forNormalized:in:)` and `viewPoint(forNormalized:in:)`
- `PreviewLayerConverter` — delegates to `AVCaptureVideoPreviewLayer.layerRectConverted`
- `PlayerLayerConverter` — aspect-fit math (Y-flip at `1 - y - h`); shipped in M2, exercised end-to-end in M3
- `OverlayStyle` — per-class colors, label format, stroke width

## Phases

### Phase 1 — Detection scaffolding & core types

Public surface only; no inference yet. `Sources/Iris/Detection/` gains
`Detector.swift`, `Detection.swift`, `TimestampedDetections.swift`. Tests use a
small `MockDetector`. The `Detector` protocol shape is locked to
`Sendable`-only per [`../DECISIONS.md`](../DECISIONS.md); stateless conformers
are `struct`, stateful conformers wrap state in an internal `actor`. Also
resolves M1-deferred `Source.state` `async` cleanup since both touch protocol
shape and land cleanest together.

### Phase 2 — `ResultStore`

`Sources/Iris/Overlay/ResultStore.swift`. `@MainActor @Observable` sorted ring
buffer; `append`, `clear`, `lookup(at: CMTime)`; staleness thresholds (500ms
live / 2s playback). Pure value-type tests (no SwiftUI, no camera). Folds in
`MockSource.bufferingPolicy:` from the M1-deferred list so multi-frame replay
tests are tractable.

### Phase 3 — Vision detector adapter

`Sources/Iris/Detection/Vision/` (subfolder for backend-specific adapters).
Wraps Vision's Swift `VNRequest` API in a `Detector` conformer — likely
`VNDetectFaceRectanglesRequest` or `VNDetectHumanBodyPoseRequest` as the first
target. `DetectorPipeline.swift` for chaining. Fixture-based test running
detection over a sample clip in `Tests/IrisTests/Fixtures/`.

### Phase 4 — `NormalizedGeometryConverting` + converters

`Sources/Iris/Overlay/NormalizedGeometryConverting.swift`,
`PreviewLayerConverter.swift`, `PlayerLayerConverter.swift`. Value types where
possible; `PreviewLayerConverter` needs `@MainActor` access to the preview
layer (see Risks). Unit tests cover aspect-fit math directly so the
`PlayerLayerConverter` half ships with coverage even though end-to-end
playback waits for M3.

### Phase 5 — `DetectionLayer` view + frame-sync

`Sources/Iris/Overlay/DetectionLayer.swift`. SwiftUI `Canvas` driven by
`TimelineView(.animation(minimumInterval: 1.0/60))`, `.drawingGroup()`,
`.allowsHitTesting(false)`. Reads `resultStore.lookup(at: displayTime)`,
converts rects via an injected converter, draws boxes + labels.

### Phase 6 — Styling & static previews

`OverlayStyle.swift`. `#Preview` cases with synthetic detections + a static
`videoRect`. Static HTML reference for box rendering — per the favorite-pattern
instinct from [`../../CLAUDE.md`](../../CLAUDE.md), visual artifacts get caught
in isolation at the same sizes/contexts they ship in.

### Phase 7 — End-to-end smoke

Live camera → Vision adapter → `ResultStore` → `DetectionLayer` on the existing
`Apps/IrisDemo-iOS` app. Smoke test on a physical iOS device asserts boxes
overlay correctly at the host frame rate.

## M1-deferred items folded in

From the M1 close entry in [`../LOG.md`](../LOG.md):

- Make `Source.state` `async` on the protocol so `CaptureSession` no longer
  needs `@preconcurrency` on conformance. → **Phase 1.**
- Add a `bufferingPolicy:` parameter to `MockSource`. → **Phase 1 or 2** —
  unblocks better `ResultStore` and pipeline tests.
- Add an iOS-targeted `xcodebuild` step to CI's test job so strict-concurrency
  bugs in `#if os(iOS)`-gated files get caught. → **Phase 1 follow-up** (small
  CI-only change; folding in early earns its keep across all later phases).
- `SampleBufferRouter` per-frame orientation lookup vs `rotationCoordinator`
  (plumbed but unused) — **defer to M3** unless detection accuracy actually
  needs per-frame orientation tagging.
- `AVCaptureSession` interruption recovery (`TODO M1+` in
  `CaptureSession.configureSession`) — **defer to M3**; not on the M2 critical
  path.

## Risks / amendments needed

- **`@unchecked Sendable` on `Detection`.** If `Detection` ends up wrapping
  non-`Sendable` Vision observation types, it needs either a documented
  invariant + `NSLock` (per the strict-concurrency escape-hatch decision in
  [`../DECISIONS.md`](../DECISIONS.md)) or a value-type unwrap into plain
  Swift structs at the adapter boundary. Phase 3 resolves which.
- **`PreviewTarget` strict-concurrency scar carries forward.** M1's shipped
  amendment dropped `@MainActor` from the protocol declaration and pushed it
  onto `setSession`. `PreviewLayerConverter` faces the analogous question —
  calling `layerRectConverted(...)` needs main-actor access. Land the
  [`explorations/display-pipeline-architecture/RECOMMENDATIONS.md`](../../explorations/display-pipeline-architecture/RECOMMENDATIONS.md)
  amendment in the same commit as Phase 4 so the recommendation doc reflects
  what shipped.
- **`displayTime` mechanism for live capture** — locked design says "host
  clock" but the timer choice (`Timer.publish`, `DispatchSourceTimer`,
  `TimelineView.now`) isn't specified. Default to `TimelineView`'s own clock
  since `DetectionLayer` is already driven by it; revisit if jitter at 60 Hz
  is visible.
- **`playerLayer.videoRect` is M3 territory.** `PlayerLayerConverter` ships
  its math in M2 but won't be exercised end-to-end until M3 adds playback.
  Unit tests cover the math directly so M2 doesn't ship dead code.
- **Stateful-actor conformer ergonomic shape is unexercised in M2.** The first
  Vision target (face rectangles or body pose) is stateless. The first stateful
  conformer (trajectory tracking, when needed) should validate that the
  actor ↔ pipeline boundary doesn't impose awkward call-site overhead. Until
  then the [`../DECISIONS.md`](../DECISIONS.md) shape (Sendable protocol;
  stateful conformers wrap state in an internal `actor`) is the contract.

## Exit criteria

- All public types listed in §Public surface compile under Swift 6 strict
  concurrency without `@preconcurrency` on the public surface.
- Vision adapter runs against a fixture video in tests; detections land in
  `ResultStore` with correct timestamps.
- `IrisDemo-iOS` renders live detection overlays on a physical iOS device at
  the host frame rate.
- M1's `Source.state` `async` cleanup landed; the `@preconcurrency Source`
  conformance on `CaptureSession` removed.
