# M8 — Image: run detectors on a single static image, swap/compare models in that one world

<!-- Working plan. Lifetime ~ this milestone; LOG.md keeps the trail. Status vocab per WORKFLOW.md §"Status trees". -->
_Defined · 2026-05-29_ · **🗓 P1–P6 pending · P1 = extract `DetectionRunner` (the source-agnostic core) next**

## Scope / intent

Run Iris's detectors on a **single static image** — an interesting captured or
playback frame, a screen capture, or any still on disk — and **swap/compare
models on that one image**. The recurring want: grab an interesting frame (or a
screenshot) and "play with different models in that world." A still is a fixed
canvas with no time axis; the value is the model-swap loop *over the same
pixels*, not motion.

The feature is **Image** throughout (a still loaded into the inspector). It
delivers a new `Sources/Iris/Image/` library folder + a demo Image page on iOS
and macOS, and ties back into capture/playback (freeze-from-live) and the M7
dataset loop (flag → export an analyzed still).

## The architecture is already source-agnostic (stated up front)

M8 is **not fighting the architecture** — the detection + overlay halves of Iris
have no video coupling, and most of the model-swap machinery is already reusable.
What forced the deferral ("don't pre-split until a second consumer lands") is now
satisfied: **the image inspector IS that second consumer.** Concretely:

- **`Frame` is source-agnostic** ([`Frame.swift`](../../Sources/Iris/Frame.swift)):
  pixelBuffer + timestamp + orientation + source + format + dimensions. Nothing
  about it presumes a stream or an asset.
- **The detection pipeline has no stream/playback coupling.**
  `Detector.detect(in: Frame) -> [Detection]` and
  [`DetectorPipeline`](../../Sources/Iris/Detection/DetectorPipeline.swift)`.detect(in:cache:tuning:)`
  take one `Frame` and return detections — already one-shot-shaped.
- **The entire overlay stack is image-agnostic.** `VideoGeometry`,
  `DetectionLayer`, `VideoRectAligned`, and `ResultStore` work by feeding
  `contentSize = imageSize` plus a frozen `displayTimeSource` — no scrubber, no
  PTS, no player layer required.
- **The model-swap machinery is fully reusable**: `DetectorCatalog`,
  `DemoModelStore`, `ActiveDetectorSession`, the tuning sheet, `RecentDetectors`.
- **The ONLY video-coupled cluster** is `PlaybackDetectionCoordinator` +
  `PlaybackController` + `PlaybackSource` + `Scrubber` + the `FlaggingSource`
  protocol — all of which assume PTS / `seek`. M8 sidesteps that cluster rather
  than reshaping it.

So M8 triggers the **source-agnostic `DetectionRunner` decomposition** the board
already deferred (the coordinator's P4 / the standing open question): lift the
loop + cache + metrics + detector-session-swap core out of the playback
coordinator into `Detection/`, and compose it from both the playback coordinator
and the new image coordinator.

## Decisions locked (2026-05-29)

- **Image detection is ONE-SHOT, not stream-driven.** A still has no time axis;
  reuse [`DetectorPipeline`](../../Sources/Iris/Detection/DetectorPipeline.swift)
  directly with a **frozen timestamp**. Do **not** route a 1-frame
  `AsyncStream<Frame>` through the playback streaming loop — that would fork the
  API shape to fit both contexts, exactly the anti-pattern CLAUDE.md forbids
  (don't fork a type's shape to serve two sources; compose instead).
- **P1 does a FULL `DetectionRunner` extraction** into
  [`Sources/Iris/Detection/`](../../Sources/Iris/Detection/) — loop + cache +
  metrics + detector-session-swap — composed by **both** the playback coordinator
  and the new image coordinator. This **resolves the long-standing
  "source-agnostic decomposition" open question** (the image inspector is the
  second consumer that justifies the split).
- **New library folder → `Sources/Iris/Image/`.** Feature named **Image**, not
  "Stills" — `SourceKind.image`, `ImageDetectionCoordinator`. (A loaded still is
  described as "a still image"; the feature/types say "Image".)
- **Freeze-from-live handoff is in scope** (P5): freeze the visible playback
  (and capture, iOS) `Frame` and open it in the Image page.
- **Dataset tie-in is in scope** (P6): it needs an **image-shaped
  `AssetFingerprint`** (drop `durationSeconds`; keep `byteSize` + head-hash) and a
  **PTS/seek-free `FlaggingSource` path** for a single still.

## Phases

### P1 — Extract `DetectionRunner` ([`Sources/Iris/Detection/`](../../Sources/Iris/Detection/))
Lift the source-agnostic core — the detect **loop + `ResultStore` cache +
`DetectionMetrics` + detector-session-swap** — out of
[`PlaybackDetectionCoordinator`](../../Sources/Iris/Playback/PlaybackDetectionCoordinator.swift)
into a `DetectionRunner` in `Detection/`. Rewire the playback coordinator to
**compose** it (the coordinator keeps the playback-coupled parts: the
`PlaybackController` + the `onDetectorTierChange → seek` pause-emit hook).
- **Acceptance:** a **pure refactor** — the full test suite (**244**) stays
  green, no behavior change; the playback detector-swap regression test still
  passes against the recomposed coordinator. **Closes the source-agnostic-
  decomposition open question.**

### P2 — Image → `Frame` ([`Sources/Iris/Image/`](../../Sources/Iris/Image/))
- Add a `SourceKind.image` case.
- A `CGImage` / `CVPixelBuffer` → `Frame` helper that **resolves EXIF /
  orientation upstream**, so the frame reaches the overlay already upright and
  `VideoGeometry` stays rotation-free (per the 2026-05-25 "VideoGeometry is the
  single coordinate authority; orientation handled upstream" doctrine).
- Fixture-image unit tests (`Tests/IrisTests/Image/`).
- **Acceptance:** a still loaded from disk becomes an **upright `Frame`** whose
  detections land correctly under the existing overlay (`VideoGeometry` +
  `DetectionLayer`).

### P3 — `ImageDetectionCoordinator` ([`Sources/Iris/Image/`](../../Sources/Iris/Image/))
`@MainActor @Observable`; owns the loaded image + a `ResultStore` +
`ActiveDetectorSession`. **One-shot detect on load**; a model swap **re-runs
detection once** and invalidates/repopulates the store (no streaming loop).
**Composes P1's `DetectionRunner`** for the loop/cache/metrics/session-swap core.
- **Acceptance:** load image → detect → swap model → re-detect, overlay redraws.
  Library-level test (no UI).

### P4 — Demo Image page (iOS + macOS)
- New `TabView` / sidebar entry for the Image page.
- An image picker for **png / jpeg / heic** UTIs (mirror
  [`DocumentPicker`](../../Apps/Shared/) — the video picker's shape).
- A `RecentImages` MRU (mirror
  [`RecentVideos`](../../Apps/Shared/RecentVideos.swift)).
- Mount the Image + `DetectionLayer` + the existing detector picker + the tuning
  sheet. **No scrubber, no seek.**
- **Acceptance:** pick an image, see detections, swap models live, on **both
  platforms**; `xcodebuild` green for both schemes.

### P5 — Freeze-from-live handoff
An **"Inspect frame"** affordance on the playback overlay (and the capture
overlay on iOS) that **freezes the current `Frame`** and opens it in the Image
page for model-play.
- **Acceptance:** from a playing or paused video, freeze the visible frame and
  run other detectors on that still.

### P6 — Dataset tie-in (flag + export an analyzed image)
- An **image-shaped `AssetFingerprint`** variant — **drop `durationSeconds`**;
  keep `byteSize` + head-hash (a still has no duration). Decide whether it shares
  a type with the video fingerprint or is a sibling (see Opens).
- A **PTS/seek-free `FlaggingSource` path** for a single still (the existing
  `FlaggingSource` assumes `currentPTS` + `seek`; an image has neither).
- The flag affordance + **PNG export** + a **dedup-ledger filename entry** for
  images — consistent with M7's "filenames are the ledger" doctrine
  ([[dataset-filenames-are-the-ledger]]; [`features/M7.md`](./M7.md) P3).
- **Acceptance:** flag an analyzed image → it exports as a **provenance-named
  PNG** into the dataset, deduped by filename.

## Owns / out of scope

- **Owns:** `Sources/Iris/Detection/DetectionRunner` (extracted source-agnostic
  core); `Sources/Iris/Image/` (image→`Frame` helper, `SourceKind.image`,
  `ImageDetectionCoordinator`); the image-shaped `AssetFingerprint` variant + the
  PTS/seek-free `FlaggingSource` path; demo wiring — the Image page, image
  picker, `RecentImages`, the freeze-from-live affordance, and the image flag /
  PNG-export path.
- **Out:** any change to the playback streaming loop's external behavior (P1 is a
  pure refactor); a 1-frame-stream image path (rejected — one-shot instead);
  multi-image / batch / contact-sheet comparison (a still at a time for v1).

## Opens / risks

- **Shared vs. sibling `AssetFingerprint`.** Does the image fingerprint share the
  video type (with `durationSeconds` made optional / dropped) or become a sibling
  struct? Decide in P6 — don't conflate the two identities by accident.
- **HEIC orientation edge cases.** EXIF orientation on HEIC/JPEG must be resolved
  upstream (P2) so `VideoGeometry` stays rotation-free; HEIC's container quirks
  are the likely footgun. Validate against a rotated-fixture image in P2.
- **`RecentImages` vs `RecentVideos` — shared MRU abstraction?** Two near-identical
  UserDefaults-backed MRUs (one with security-scoped bookmarks, one possibly
  without) invite a shared generic. Decide in P4 whether to factor a common base
  or keep them as deliberate siblings.
