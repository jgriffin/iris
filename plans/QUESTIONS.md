## Answered

- рџ“Њ [answered 2026-05-26] **Runtime-tunable Core ML detector thresholds.** Should a Core ML `CoreMLDetector` expose its detection thresholds as runtime `TunableDetector` knobs? The catch is path-dependent: Path A (`nms=True`, the YOLOv12 zero-decode path) **bakes the IoU/confidence thresholds into the exported model at export time**, so runtime-tunable thresholds would force **Path B** (raw tensor + a Swift threshold/NMS decoder) or a **re-export per threshold set** вЂ” i.e. it changes *which `OutputDecoder` the YOLO catalog entries use*. P2 settled it **for P2** by shipping the Path-A `VisionObjectDecoder` with baked (export-time) IoU/conf thresholds and conforming `CoreMLDetector` to `Detector` only (no runtime knobs вЂ” no fake `TunableDetector`); the genuinely-runtime-tunable variant is **re-scoped to M6В·P3**, since it requires a path-B raw-tensor decoder + Swift NMS/threshold and is decided alongside `YOLOEnd2EndDecoder`. See [`DECISIONS.md`](./DECISIONS.md) and [`features/M6.md`](./features/M6.md) В§"Open design questions".

- рџ“Њ [answered 2026-05-26] **iOS/macOS 26 auto-decode observation type.** The exact value-type observation name for an auto-decoded Core ML detector under the new `CoreMLRequest` API (vs. legacy `VNRecognizedObjectObservation`) was confirmed at runtime in P2 against the MacOSX26.5 SDK: `CoreMLRequest.Result == [any VisionObservation]`, and an NMS object-detection pipeline yields concrete **`RecognizedObjectObservation`** (logged via a temporary probe). Not a formal QUESTIONS entry while open (the рџ‘Ђ lived in [`features/M6.md`](./features/M6.md)); recorded here for the trail. See [`DECISIONS.md`](./DECISIONS.md).

- рџ“Њ [answered 2026-05-23] **Cache fingerprinting strategy for `ResultStore` under M4.** Per-entry fingerprinting was the wrong shape вЂ” the cache is *globally* a record of "what the current detector configuration produced", not a heterogeneous mix of per-setting snapshots. The Phase 2 `invalidateAll()` on detector-tier change IS the global model and was already correct. User pushback caught it (work block 4 of 2026-05-23): a 24-byte fingerprint Г— 108k entries with per-knob inequality checks was inventing complexity to solve a non-problem. The genuinely interesting alternative вЂ” running the detector at maximally-permissive settings so all threshold knobs become filter-tier by construction вЂ” was investigated (work block 5) and found not free: cheap for shape knobs (aspect, size, quadrature), bad for `minimumConfidence` and `maximumObservations` (garbage-output explosion + cache bloat). Deferred indefinitely; the paused-frame re-emit hook + working filter projection make detector-tier changes feel fast enough in practice. See work blocks 4вЂ“5 of 2026-05-23 in [`LOG.md`](./LOG.md).

- рџ“Њ [answered 2026-05-23] **Settings persistence вЂ” `TuningModel` vs. consumer responsibility.** Consumer-concern, as suspected. - рџ“Њ [answered 2026-05-22] **`displayTime` semantic divergence between capture (host clock) and playback (asset time), and the exact-equality assumption in `ResultStore.lookup`.** Resolved by [`features/playback-detection-cache.md`](./features/playback-detection-cache.md) вЂ” `ResultStore` is now a quantized-timestamp-keyed dictionary cache with nearest-neighbor `lookup(at:)` within `min(2 Г— quantization, stale:)`. The per-source `Frame.timestamp` contract still differs (capture = host clock, playback = asset time), but lookup no longer assumes exact equality, so the divergence stops mattering at the millisecond scale that tripped up real playback. The originally-suspected exact-equality assumption turned out to be a *minor* factor; the load-bearing failure was actually `ResultStore`'s 30-frame ring buffer evicting most playback history, and the lack of detection-result reuse across re-visits. Both fixed in the same work. See [`DECISIONS.md`](./DECISIONS.md) В§"Best-effort temporal match in `ResultStore.lookup`".
- рџ“Њ [answered 2026-05-21] **`Detector` stateful-conformer shape.** Protocol stays `Sendable`-only; stateful conformers wrap state in an internal `actor` by their own choice.
