- Learned: the locked `@MainActor public protocol PreviewTarget` design in [`display-pipeline-architecture/RECOMMENDATIONS.md`](../explorations/display-pipeline-architecture/RECOMMENDATIONS.md) §Type sketches doesn't satisfy Swift 6 strict concurrency — the `@MainActor`-pinned existential can't be sent into a `Task`. Working amendment: drop `@MainActor` from the protocol declaration, push it onto `setSession` and onto `PreviewSource.connect(to:)`. Caller (`CameraPreview.makeUIView`) is already `@MainActor` via `UIViewRepresentable`, so the internal `Task` hop drops out entirely.
- Learned: `swift build` on the macOS host never compiles `#if os(iOS)` files, so it doesn't catch strict-concurrency bugs in the Capture folder. `xcodebuild -destination 'generic/platform=iOS'` (or a real `xcodebuild build`) is required to validate the iOS-gated subset. Worth adding to CI's test job.
- Learned: xcodegen auto-fills `PRODUCT_BUNDLE_IDENTIFIER = <bundleIdPrefix>.<target-name>` as an INLINE target setting when the target has no explicit value — sitting at higher precedence than xcconfig. Fix: drop `bundleIdPrefix` from `project.yml` so xcconfig wins cleanly.
- Learned: `CaptureSession` conforms via `@preconcurrency Source` because the actor's `var state` can't synchronously satisfy the protocol's nonisolated `get`. Clean long-term fix is making the `Source` protocol declare `var state: SourceState { get async }`. Punted to M2.
- Learned: `MockSource.start()` yields all supplied frames synchronously, and `.bufferingNewest(1)` means a single consumer only sees the LAST frame. Tests had to run one frame per session to assert ordering. Consider letting `MockSource` take a `bufferingPolicy:` parameter in M2.
